# 仿写网页

**目标**：先**学习**别人网页的结构、样式和组件，再**仿写**出用户想要的那一页。

工具：[WebTwin](https://github.com/sirioberati/WebTwin) —— 把目标网站抓成 HTML/CSS/JS/图片等，打包成 ZIP，供你阅读和学习。

---

## 整体流程（两步）

1. **学习参考网页**：用 WebTwin 把「要模仿的网站」抓下来，得到 ZIP；解压后阅读其中的 HTML/CSS/JS，总结布局、配色、组件、字体等。
2. **仿写用户要的网页**：根据学到的内容 + 用户对「我要的网页」的描述，在当前项目里写出新页面（可复刻风格、改文案、改结构）。

---

## 第一步：用 Git 里的 WebTwin 抓取参考站

### 1.1 拿到 WebTwin（若还没有）

```bash
git clone https://github.com/sirioberati/WebTwin.git
cd WebTwin
```

### 1.2 安装依赖并启动

```bash
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

### 1.3 在浏览器里抓站

- 打开 **http://127.0.0.1:5001**
- 输入**要学习的参考站 URL**
- 若是普通静态站：直接点「Extract Website」
- 若是 React/Vue/Angular/Next 等 **JS 多的站**：勾选「Use Advanced Rendering (Selenium)」再点「Extract Website」
- 等完成后**下载生成的 ZIP**

### 1.4 解压并交给 AI 学习

- 把 ZIP 解压到当前项目下的一个目录（例如 `reference_site/`）
- **阅读**解压后的 HTML、CSS、JS 文件，总结：
  - 整体布局（header/footer/主内容区）
  - 配色、字体、间距
  - 关键组件（导航、卡片、按钮、表单等）的 HTML 结构和 class 命名
  - 用到的图片、图标、字体资源路径

这些总结就是「从别人网页学到的东西」，用于第二步仿写。

---

## 第二步：仿写用户要的网页

- 根据**用户对目标页的描述**（例如：首页、落地页、带表单的注册页、某竞品风格的介绍页等）
- 结合**第一步学到的**布局、样式、组件，在当前项目里：
  - 新建或修改 HTML/模板/组件
  - 复用或改写参考站的结构与风格，替换成用户要的文案、链接、功能
  - 如需用 Tailwind/React/Vue 等，在保持「仿写风格」的前提下按项目技术栈实现

**输出**：用户要的那一页的代码（及所需样式/资源说明），可直接在当前项目中使用。

---

## 何时用高级渲染（Selenium）

- 参考站是 **SPA 或大量 JS 渲染**（React、Vue、Angular、Next.js、Nuxt 等）→ 必须勾选「Use Advanced Rendering (Selenium)」
- 否则可能只拿到空壳 HTML，学不到真实布局和样式
- 需要本机已安装 **Chrome/Chromium**，WebTwin 会用其做无头渲染

## 给 OpenClaw / Agent 的用法小结

1. **用户说「仿写某网页」或「参考某站做一页」** → 启用本 skill。
2. **若本机还没有 WebTwin**：按上面 1.1～1.2 克隆、安装、运行；若已有，则直接 `cd WebTwin && source venv/bin/activate && python app.py`。

5. **根据用户对「我要的网页」的描述**，在当前项目里写出仿写后的页面代码，风格和结构参考学到的内容。

## 用户给出参考站后：要模仿哪些方面的框架（方法论）

**每次**用户提供「要模仿的网站」URL 后，你应针对**该站**做抓取与阅读，并按下述维度做总结，再按这些维度去仿写。不是套用固定模版，而是「从用户给的参考站里学什么、仿什么」的框架。

### 要从参考站学习的维度

1. **整体页面结构**
   - 从上到下有哪些大区块（导航、首屏/Hero、轮播/列表、产品演示、Footer 等）。
   - 是否有侧栏（如内容导览）、主内容几栏、是否 sticky。

2. **导航与信息架构**
   - 主导航项、层级、主 CTA 按钮位置与文案。
   - 是否有二级/侧栏导航，当前项如何高亮。

3. **首屏 / Hero**
   - 标题层级、副标题、是否多段文案、是否有标签/按钮。
   - 背景（纯色/渐变/图）、是否全屏、排版居中或左对齐。

4. **核心内容组件（按参考站实际有为准）**
   - 若有轮播：每屏布局（左文右图或上下）、箭头与圆点、切换方式；从 HTML/CSS 里看出 track、slide、dot 的结构与 class。
   - 若有步骤/工作流：左侧步骤结构（图标+标题+描述）、右侧配图或录屏、是否随步骤切换图。
   - 若有产品视频：容器样式、封面、播放交互（点击封面或按钮后播、是否隐藏封面）。

5. **配色与字体**
   - 主色、背景色、深色块用在哪（整页还是仅 footer/单块）。
   - 字体族、标题与正文字号/字重。

6. **细节与交互**
   - 圆角、阴影、边框风格；按钮/链接 hover 态。
   - 轮播、Tab、视频等交互在参考站里如何实现（DOM 结构、class、是否用 JS 切换）。

### 仿写时怎么做

- 用上述维度的总结，把**用户要给的内容**（文案、图、视频路径）填进**从参考站学来的结构**里。
- 不预设「常见模版」；每次都以用户给的参考站为准，按维度去学、去还原；若抓不到完整 HTML，可用 mcp_web_fetch 看文案与区块，再按维度补结构。

## 注意

- 仅用于学习、参考或已获授权的场景；遵守目标站的使用条款与 robots.txt。
- 部分站点有反爬或需登录，可能抓不全；大站可只抓单页或分多次抓。

## Troubleshooting（常见问题）

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 抓取结果只有空壳 HTML | 目标站是 SPA/大量 JS 渲染 | 勾选「Use Advanced Rendering (Selenium)」再抓 |
| WebTwin 启动失败 | 依赖未装或端口占用 | 确认 venv 已激活、`pip install -r requirements.txt`、必要时改端口 |
| 仿写风格不像参考站 | 未充分阅读解压后的 CSS/组件 | 先总结参考站配色、字体、组件 class 再写 |
| 目标站无法访问 | 网络/墙/robots 限制 | 换网络或只仿写用户提供的截图/描述，注明未抓取到现场 |
